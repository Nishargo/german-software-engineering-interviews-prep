# Technisches Interview

Das technische Interview prüft nicht nur theoretisches Wissen, sondern vor allem praktische Erfahrung, Systemverständnis und Ihre Fähigkeit, reale Szenarien zu erklären.

Oft basieren die Fragen direkt auf Ihrem Lebenslauf und zielen darauf ab, zu verstehen, **was Sie konkret getan haben und warum bestimmte Entscheidungen getroffen wurden**.

---

## Fragen zu Ihrem Lebenslauf & Projekten

* Erzählen Sie mir Ihr letztes Projekt im Detail.
* Was genau war Ihre Aufgabe in diesem Projekt?
* Welche konkreten Features haben Sie implementiert?
* Wie sah Ihre tägliche Arbeit aus?
* Welche Entscheidungen haben Sie selbst getroffen?
* Wo waren Sie nur Implementierer und wo haben Sie Architekturentscheidungen getroffen?
* Was war das komplexeste Problem in Ihrem Projekt?
* Wie haben Sie dieses Problem gelöst?
* Was würden Sie heute anders machen?

---

## Systemverständnis aus Ihrem Projekt

* Wie funktioniert der komplette Ablauf Ihrer Anwendung?
* Was passiert Schritt für Schritt, wenn ein User sich einloggt?
* Wie wird ein Request vom Frontend bis zum Backend verarbeitet?
* Wie funktioniert Authentication in Ihrem System?
* Was passiert, wenn ein User eine geschützte Route aufruft?
* Wie funktioniert JWT in Ihrem System konkret?
* Wo wird das Token gespeichert und warum?
* Was passiert, wenn das Token abläuft?
* Wie wird Authorization umgesetzt?

---

## CI/CD & DevOps 

* Haben Sie CI/CD in Ihrem Projekt verwendet?
* Was passiert genau in Ihrer CI/CD Pipeline?
* Welche Schritte passieren bei einem Commit?
* Was passiert bei einem Build-Fehler?
* Was passiert bei einem failed Test?
* Wie wird Code von Development bis Production deployed?
* Haben Sie Blue-Green oder Rolling Deployments verwendet?
* Was passiert, wenn ein Deployment fehlschlägt?
* Wie würden Sie eine Pipeline verbessern?

---

## HTTP & Request Flow 

* Was passiert, wenn ein User einen API Call macht?
* Wie wird ein HTTP Request im Backend verarbeitet?
* Was passiert bei einer 401 vs 403 in Ihrem System?
* Wie gehen Sie mit Fehlern im Backend um?
* Was passiert bei einem Timeout?
* Wie würden Sie Logging und Monitoring implementieren?

---

## Datenstrukturen im echten Kontext

* Wo haben Sie Arrays in Ihrem Projekt verwendet?
* Wo haben Sie Datenbanken statt In-Memory-Strukturen genutzt?
* Warum haben Sie diese Datenstruktur gewählt?
* Welche Performance-Probleme sind Ihnen begegnet?

---

## AWS / Cloud 

* Welche AWS Services haben Sie konkret genutzt?
* Wie wurde Ihre Anwendung deployed?
* Wie ist Ihre Architektur aufgebaut (Frontend, Backend, DB)?
* Wie wird Skalierung in Ihrem System erreicht?
* Was passiert bei hoher Last?
* Wie würden Sie Ihr System hochverfügbar machen?

---

## Docker & Containerisierung 

* Haben Sie Ihre Anwendung containerisiert?
* Wie sieht Ihr Docker Setup aus?
* Was passiert beim Build eines Containers?
* Wie kommunizieren mehrere Container miteinander?
* Wie debuggen Sie einen laufenden Container?
* Warum haben Sie Docker in Ihrem Projekt verwendet?

---

## Szenario-basierte Fragen

* Wie würden Sie Feature X in Ihrem bestehenden System hinzufügen?
* Was passiert, wenn plötzlich 10x mehr User kommen?
* Wie würden Sie einen Performance-Bottleneck analysieren?
* Was passiert, wenn die Datenbank ausfällt?
* Wie würden Sie Logging und Monitoring verbessern?
* Wie würden Sie ein bestehendes System refactoren?

---

## Fragen basierend auf Ihrer Stelle

* Haben Sie genau diese Technologie schon einmal in einem Projekt verwendet?
* Wie würden Sie sich in diese Technologie einarbeiten?
* Was wäre Ihr Ansatz, wenn Sie diese Aufgabe bekommen?

---

## Eigene Fragen an das Unternehmen

* Wie sieht eure Architektur im Detail aus?
* Wie funktioniert euer Deployment-Prozess?
* Welche CI/CD Tools nutzt ihr?
* Wie wird Authentifizierung bei euch umgesetzt?
* Welche größten technischen Herausforderungen habt ihr aktuell?
